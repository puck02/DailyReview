# AI 辅助学习 Web 应用设计规格

## 目标

在当前 VPS 上开发一个面向 1-5 人的小型 AI 辅助学习 Web 应用。系统提供类似 ChatGPT 网页端的问答体验，支持文字、图片上传和剪贴板粘贴图片，并基于每日问答内容生成个人学习报告。

应用优先轻量、可维护、低资源占用。当前服务器约束为 2 核 CPU、1.8GiB 内存、剩余磁盘约 7.7G，因此第一版不引入 Postgres、Redis、Celery、独立向量库或多容器架构。

## 已确认范围

- 使用账号体系，每个用户数据隔离。
- 注册采用邀请制，普通用户注册必须提供有效邀请码。
- 管理员只负责生成和查看邀请码状态，不查看其他用户聊天和报告。
- 聊天支持最近 7 天多会话列表，可继续上下文。
- 聊天支持文字、图片文件上传、剪贴板图片粘贴。
- AI 回复使用流式输出。
- 默认模型为 `gpt-5.4-mini`，用户可手动切换到复杂模型 `5.5`。
- 每天 23:00 按用户生成个人日报。
- 每周日生成个人周报，每月最后一天生成个人月报。
- 聊天会话、消息、上传图片只保留 7 天。
- 日报、周报、月报长期保留。
- 图片不长期写入报告引用，报告中只保留图片相关讨论的文字化概述。

## 技术方案

推荐采用单体轻量架构：

- 后端：FastAPI
- 数据库：SQLite
- 前端：Vite + React
- 生产托管：前端构建为静态文件，由 FastAPI 静态托管
- 定时任务：后端进程内使用 APScheduler
- 文件存储：本地 `data/` 目录
- AI 接入：OpenAI-compatible API，通过环境变量配置

生产运行只需要一个 Python 后端进程，不常驻 Node 服务。

## 配置与敏感信息

真实 API Key、管理员初始密码和服务地址不写入代码、文档或提交记录。使用以下环境变量：

- `AI_BASE_URL`
- `AI_API_KEY`
- `AI_DEFAULT_MODEL=gpt-5.4-mini`
- `AI_COMPLEX_MODEL=5.5`
- `ADMIN_EMAIL`
- `ADMIN_INITIAL_PASSWORD`
- `SECRET_KEY`
- `DATABASE_URL`
- `APP_TIMEZONE=Asia/Shanghai`

首次启动时，如果数据库中没有管理员用户，后端根据 `ADMIN_EMAIL` 和 `ADMIN_INITIAL_PASSWORD` 初始化管理员账号，并只保存密码哈希。

## 数据模型

SQLite 表设计：

- `users`
  - 用户邮箱、密码哈希、角色、创建时间。
- `invite_codes`
  - 邀请码、创建者、使用者、是否已使用、过期时间、创建时间。
- `chat_sessions`
  - 所属用户、标题、默认模型、创建时间、更新时间。
- `messages`
  - 所属会话、角色、文本内容、模型、创建时间。
- `attachments`
  - 所属消息、文件路径、MIME 类型、大小、过期时间、创建时间。
- `reports`
  - 所属用户、报告类型、周期日期、Markdown 路径、统计 JSON、创建时间。

不创建向量表或内容块表。历史内容关联在生成报告时直接读取 Markdown 文件，按标题和段落切块，通过本地文本检索召回相关段落。

## 报告生成

日报按用户独立生成，默认时区为 `Asia/Shanghai`。

日报流程：

1. 每天 23:00 触发。
2. 读取该用户当天 00:00-23:00 的聊天内容。
3. 如果当天没有聊天内容，不生成日报。
4. 提取当天主题、问题、已掌握内容、仍不清楚的点。
5. 读取过去 7 天日报 Markdown。
6. 从当天内容中提取 3-8 个关键词，在历史报告 Markdown 中做本地文本检索。
7. 截取命中段落及相邻段落作为历史上下文。
8. 调用模型生成 Markdown 日报。
9. 写入 `data/reports/user-<id>/daily/YYYY/MM/YYYY-MM-DD.md`。
10. 在 `reports` 表保存索引和统计 JSON。

日报结构借鉴 NovaForge 的 7 步结构，并适配学习复盘：

```md
# YYYY-MM-DD 学习日报

## 1. 今日学习概览
## 2. 核心知识点
## 3. 典型问题与解法
## 4. 易错点 / 未解决问题
## 5. 与历史内容的关联
## 6. 下一步建议
## 7. 简短复盘
```

日报长度控制在约 5 页以内，输出应简洁，必须包含下一步建议。

周报与月报：

- 每周日日报生成后，再生成本周周报。
- 每月最后一天日报生成后，再生成月报。
- 如果同一天既是周日又是月底，执行顺序为：日报、周报、月报。
- 周报和月报只读取已生成的日报，不读取 7 天前的原始聊天。
- 前端可视化基于报告统计 JSON，例如主题分布、问题数量、连续学习天数、待解决问题数量。

## 数据保留与清理

每天报告任务结束后执行清理任务：

- 删除 7 天前的聊天会话。
- 删除对应消息和附件索引。
- 删除 7 天前的上传图片文件。
- 不删除日报、周报、月报。
- 不删除用户和邀请码记录。

## 前端页面

### 登录 / 注册

- 登录：邮箱、密码。
- 注册：邮箱、密码、邀请码。
- 邀请码无效、已使用、过期时给明确提示。

### 聊天页

- 左侧显示最近 7 天会话列表。
- 中间显示消息流。
- 底部包含输入框、图片上传按钮、模型选择器。
- 支持粘贴图片并附加到当前消息。
- AI 回复以流式方式追加显示。
- 会话标题可由首条问题自动生成。

### 报告页

- 月份选择器。
- 日报列表。
- Markdown 预览。
- 周报和月报入口。
- 只显示当前用户自己的报告。

### 管理员页

- 仅管理员可访问。
- 可生成邀请码。
- 可查看邀请码状态：未使用、已使用、过期。
- 不显示用户聊天或报告。

## API 分组

- `auth`
  - 登录、注册、退出、获取当前用户。
- `invites`
  - 管理员生成邀请码、查看邀请码列表。
- `sessions`
  - 创建会话、列出最近 7 天会话、更新标题、删除会话。
- `chat`
  - 发送消息、上传图片、流式获取 AI 回复。
- `reports`
  - 按月份列出日报、读取日报、读取周报、读取月报、获取统计数据。

## 权限边界

- 普通用户只能访问自己的会话、消息、附件和报告。
- 管理员权限仅扩展到邀请码管理。
- 所有非公开接口都校验登录状态。
- 用户密码只保存哈希。
- API Key 只存在服务端环境变量中，不返回前端。

## 开发里程碑

1. 项目初始化
   - FastAPI、React、SQLite、基础配置、静态文件托管、`.env.example`。
2. 认证与邀请码
   - 管理员初始化、登录、注册、邀请码生成、权限校验。
3. 聊天 MVP
   - 会话列表、文本聊天、流式输出、模型手动切换。
4. 图片能力
   - 文件上传、剪贴板粘贴、图片随消息发送给模型、7 天清理。
5. 日报系统
   - 定时生成、NovaForge 风格 Markdown、历史报告文本检索、报告预览和月份检索。
6. 周报与月报
   - 周日和月底生成总结，前端展示简单统计图表。
7. 部署收尾
   - 生产构建、systemd 服务、数据备份说明、运行验证。

## 不在第一版范围

- 多团队或组织管理。
- 管理员查看其他用户聊天或报告。
- 向量检索、Embedding 存储、独立搜索服务。
- 邮件邀请、第三方登录。
- PDF 导出。
- 移动端原生应用。
- 复杂权限角色体系。
